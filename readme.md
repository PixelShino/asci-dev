# ASCI DEV PORTFOLIO

Интерактивный лендинг-презентация разработчика, спроектированный в стиле терминальной консоли. Проект представляет собой отказоустойчивую Fullstack-экосистему с поддержкой интернационализации (i18n), интерактивными 3D/WebGL компонентами и встроенным ИИ-сопроцессором, контролирующим фоллбеки маршрутизации.

- **Ссылка на деплой:** `https://goldobin.vercel.app/ru`
- **Репозиторий:** `https://github.com/PixelShino/asci-dev`

---

## 1. Стек технологий (Tech Stack)

### Frontend & Core

- **Фреймворк:** Next.js 16
- **Язык:** TypeScript
- **Стилизация & UI:** Tailwind CSS, Framer Motion
- **Локализация:** `next-intl` (синхронизация двуязычных схем перевода `ru`/`en`)
- **3D & Графика:** React Three Fiber (R3F), Three.js (рендеринг и интерполяция движения ASCII-модели в рантайме)

### Backend & Infrastructure

- **Рантайм среда:** Node.js, Next.js Route Handlers (Serverless-контур)
- **Телеметрия:** `@vercel/analytics/next` (real-time мониторинг показателей производительности Core Web Vitals и посещаемости )
- **Хостинг:** Vercel

## 2. Локальный запуск (Local Setup)

### Клонирование репозитория и установка зависимостей:

```bash
git clone [https://github.com/PixelShino/asci-dev.git](https://github.com/PixelShino/asci-dev.git)
cd asci-dev
npm install
```

### Настройка переменных окружения (`.env.local`):

Создайте файл `.env.local` в корне проекта и укажите следующие ключи:

```env
# API ключи для ИИ-сопроцессора
GEMINI_API_KEY="google_ai_studio_api_key"
OPENROUTER_API_KEY="openrouter_api_key"

# Конфигурация SMTP (Nodemailer) для формы обратной связи
SMTP_HOST="smtp.mail.ru"
SMTP_PORT=465
SMTP_USER="email@mail.ru"
SMTP_PASS="pass"
```

### Запуск сервера разработки:

```bash
npm run dev
```

Приложение будет доступно по адресу `http://localhost:3000`.

## 3. Реализация формы обратной связи и API

Форма спроектирована как замкнутый e2e-цикл: **Frontend → API → Обработка ошибок/Состояния → Результат трансляции**.

### Frontend часть:

- Валидация полей (`name`, `email`, `comment` — обязательные, `phone` — опциональный) реализована на клиенте с помощью контролируемых инпутов.
- Интерфейс обрабатывает **5 состояний рантайма**: `IDLE` (ожидание), `LOADING` (передача пакетов данных), `SUCCESS` (код ответа 200 OK), `ERROR` (диагностика сбоя) и `EMPTY` (блокировка пустой отправки).

### Backend / API часть (`app/api/contact/route.ts`):

- Роут принимает `POST` запросы, выполняет серверную санитарию и валидацию входящих данных.
- Инициализирует транспортный шлюз `Nodemailer`.
- **Двухпоточная отправка писем:**
  1. **Поток А:** Отправка payload-данных заявки на email владельца портфолио.
  2. **Поток Б:** Отправка официальной копии трансляции (confirmation email) на почту, указанную пользователем.
- При сбое в SMTP-транспорте бэкенд перехватывает исключение и возвращает клиенту валидный JSON-лог ошибки со статусом `500`, предотвращая падение серверного инстанса.

## 4. Интеграция AI-сопроцессора (`AiAssistant`)

В проект интегрирован ИИ-помощник ( OpenRouter и Gemini).

### Логика роутинга (`app/api/ai/route.ts`):

1. **Основной узел:** Запрос отправляется напрямую в `Google AI Studio` (`gemini-2.5-flash`).
2. **Фоллбек-режим (Режим AUTO):** Поскольку сервера Vercel работают в дата-центрах AWS (регион `us-east-1`), их IP-адреса часто превентивно блокируются брандмауэром Google AI Studio с ошибкой `denied access`. Для обеспечения 100% аптайма ассистента в режиме `AUTO` реализована логика Failover Proxy: при фиксации сбоя сети с Gemini бэкенд автоматически перенаправляет промпт на резервный шлюз `OpenRouter` (`deepseek-chat`) или другой доступный .
3. **Контекст и Язык:** В системный промпт зашита строгая языковая логика: по умолчанию ИИ отвечает на русском языке.

## 5. Использование AI в работе и ручные исправления

ИИ использовался как **помощи в написании кода, консультации по багам, и анализа ошибок** , однако финальный архитектурный надзор и исправление критических runtime-багов выполнялись исключительно вручную.

### Что доверялось ИИ:

- Генерация базовой структуры JSON-файлов локализации для `next-intl` (`ru.json`, `en.json`).
- Бойлерплейт-код для разметки стандартных UI-компонентов и Tailwind-стилей консоли.
- Первичный набросок структуры системного промпта для контекста ИИ-помощника.
- Помощь с конвертацией модели в `tsx` и в целом с дебагом модели
- Отчистки mediaViewer от логики прошлого проекта

### Что исправлялось и дорабатывалось вручную:

- **Устранение утечки памяти в `ProjectsLayout`:** Логика бесконечного скролла на базе `IntersectionObserver` вызывала дублирование карточек проектов. При переходе в детальный режим триггер оставался в DOM-дереве и обсервер накручивал стейт. Ошибка исправлена ручным внедрением условия раннего возврата `if (selectedProject) return` и принудительным сбросом стейта буфера в методе `handleBack`.
- **Исправление падения `next-intl` в `ProjectDetails`:** В статических константах проектов длина массива фичей составляла 19 элементов, в то время как обновленные JSON-схемы содержали по 3 фичи на проект. Цикл вызывал ошибку `MISSING_MESSAGE` и ронял рантайм. Баг устранен синхронизацией конфигурационных данных и внедрением защитного метода `t.has(translationKey)` для безопасного пропуска пустых ключей.
- **Обход CORS для ASCII-рендеринга иконок (`SkillIcon`):** Компонент `<AsciiImage>` попиксельно считывал данные картинок с удаленного сервера `skillicons.dev` через JavaScript для генерации символов, что блокировалось политикой CORS в браузере. Проблема решена ручным конфигурированием правил `rewrites` в `next.config.mjs`.
  Трафик замаскирован под Same-Origin запросы (`/api/icons`), а для сброса агрессивного кэширования браузера на CORS-ошибки в URL внедрен cache-buster (`&v=1`).
- **Валидация фавиконок:** Ручная сборка и маршрутизация полного пакета Favicon-ассетов через корень `public/favicon/` с явным маппингом разрешений в `Metadata` и ручной пропиской путей внутри `site.webmanifest` для корректного отображения PWA-иконок на Android/iOS.
- **Финальное тестирование локализации:** Ручной тест интерфейса при переключении языковых локалей `ru`/`en`. Полнотекстовая проверка рендеринга консольных логов и переводов ассистента, исключающая появление необработанных ключей или съезжания шрифтов.
- **Инспекция адаптивности и WebGL-слоя:** Сквозное тестирование UI-сетки на реальных мобильных устройствах и различных разрешениях дисплеев. Проверка корректности масштабирования интерактивного контейнера 3D ASCII-модели в рантайме, отладка производительности анимации без просадки FPS на мобильных CPU/GPU.
- **Smoke-тестирование после деплоя (Post-Deployment QA):** Комплексный ручной аудит сценариев непосредственно на живом сервере Vercel после публикации релиза. Проверка сетевых логов на предмет успешного обхода CORS-ограничений прокси-сервером и сквозная отправка писем через рабочую форму обратной связи.
